# 03 — UI Tree Architecture  
**呼吸式 UI の正式仕様書（Breath-Based Interface）**

UI-Tree Architecture（以下 UI-Tree）は、  
筆者とAIが日々の観察を通じて自然に形成した  
「呼吸で動くユーザーインターフェース」  
の正式定義である。

UI-Tree は次の三つで構成される：

1. **Tree（構造）**  
2. **Breath（呼吸）**  
3. **Sync（同期）**

この三要素が揃うと、  
UIはただの「操作画面」ではなく、  
“人の状態に寄り添って形を変える生き物”になる。

---

# 1. UI-Tree とは何か

UI-Tree は、  
「人間の呼吸状態を中心に枝が伸びたり縮んだりするUI」  
であり、次の特徴を持つ。

- **操作より呼吸が優先される**  
- 情報の“詰め込み”ではなく  
  “風が通る余白”が設計の中心となる  
- 思考の負荷が低く、自然な移動ができる  
- 枝構造によって“思考の動線”が可視化される  
- AI がユーザーの“揺れ”を判断して表示を最適化する  

UI-Tree は「指示ではなく、呼吸で動くUI」である。
